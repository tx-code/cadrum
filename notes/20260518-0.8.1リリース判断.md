# 0.8.1 リリース判断メモ

## 結論

`0.8.1-alpha.1` から alpha 接尾辞を外して **`0.8.1`** で publish する。

## 動機

LLM 援助のある自動 CAD 設計ループ(`Solid::write_multiview_png`)を default 構成で使えるようにすること。これは PNG 出力が `Cargo.toml` の features に依存しない、ライブラリ取り込みデフォルトで動く状態をユーザーに提供することを意味する。

## tiny-skia を default に入れることのコスト

`png` feature を default に含めると `tiny-skia` (+ 推移依存) が常に取り込まれる。事前計測:

| 項目 | 値 |
|---|---|
| 追加 crate 数 | +18 (16 → 34) |
| 追加 crate の性質 | 全て pure Rust、proc-macro なし、C 依存なし |
| Cold release build (tiny-skia ツリーのみ) | ~4.4 秒 |
| Cold debug build | ~3.3 秒 |
| Warm `cargo check` | ~2.5 秒 |
| Incremental build | 0 秒(tiny-skia は再ビルドされない) |
| .rlib 合計 (release) | ~7 MB (うち tiny-skia 本体 2.3 MB) |
| 最終バイナリ増分 (推定、LTO 後) | 0.5–1.5 MB |

OCCT 静的リンクの数十 MB / prebuilt tarball ダウンロードの数十秒に比べれば誤差レベル。

## 0.8.0 → 0.8.1 の変更サマリ

### 破壊的(API)
- `Mesh.normals: Vec<DVec3>` フィールドを削除
- `EdgeData` 型を削除、`Mesh.edges` を `Vec<...>` に変更、lib.rs から `EdgeData` の re-export を削除
- `Mesh::to_svg` / `Mesh::write_svg` を削除、`Scene2D` に移動
- `Scene2D` を新規 pub re-export

### 追加(非破壊)
- `Color::as_u16` (STL 属性バイト用 RGB555 エンコード)
- `Error::PngExportFailed`
- `Solid::write_multiview_png` (1024×1024 固定 4 視点 PNG、`png` feature gate)
- `default = ["color"]` → `["color", "png"]`
- examples: `12_multiview`, `09_bspline`, `10_fillet`, `11_chamfer`

## SemVer に関する注記

Cargo の 0.x.y ルールでは x の bump が breaking line(`cadrum = "0.8"` は `>=0.8.0, <0.9.0` 内を互換と見なす)。`Mesh.normals` / `EdgeData` の削除は厳密には minor bump (0.9.0) を要する破壊的変更。

このプロジェクトは 0.7.0 → 0.7.6 で patch bump を連発しており(`Solid::history` 導入や CxxOpaqueStruct 廃止など破壊的変更を含む)、過去から SemVer を厳密運用していない。今回も同じ慣行で 0.8.1 として出す判断。直接利用者の追従コストはあるが、LLM 自動設計ループを default で使える状態にする価値を優先する。
